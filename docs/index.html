虚拟渲染技术

传统轮播组件会渲染所有数据项，当数据量大时会造成严重的性能问题。

taro-virtual-swiper 采用虚拟渲染策略，只渲染可视区域内的元素：

  • 当前显示项
  • 前一项（用于向前滑动）
  • 后一项（用于向后滑动）

无论你的数据列表有多长，DOM 节点数量始终保持在 2-3 个，确保流畅的用户体验。

基础用法

import { VirtualSwiper } from 'taro-virtual-swiper';
import 'taro-virtual-swiper/style.scss';

const data = [
  { id: 1, title: 'Slide 1', image: '/images/1.jpg' },
  { id: 2, title: 'Slide 2', image: '/images/2.jpg' },
  { id: 3, title: 'Slide 3', image: '/images/3.jpg' },
  // ... 更多数据
];

function App() {
  return (
    <VirtualSwiper
      list={data}
      renderItem={(item, index) => (
        <View className="slide-item">
          <Image src={item.image} />
          <Text>{item.title}</Text>
        </View>
      )}
      onChange={(index, item) => {
        console.log('当前索引:', index);
      }}
    />
  );
}
3

受控模式

import { useState } from 'react';
import { VirtualSwiper } from 'taro-virtual-swiper';
import 'taro-virtual-swiper/style.scss'

function App() {
  const [current, setCurrent] = useState(0);

  const handlePrev = () => {
    setCurrent(Math.max(0, current - 1));
  };

  const handleNext = () => {
    setCurrent(Math.min(data.length - 1, current + 1));
  };

  return (
    <>
      <VirtualSwiper
        list={data}
        current={current}
        onChange={(index) => setCurrent(index)}
        renderItem={(item) => <View>{item.title}</View>}
      />
      <Button onClick={handlePrev}>上一个</Button>
      <Button onClick={handleNext}>下一个</Button>
    </>
  );
}

API

属性 类型 默认值 说明
list {id: string|number, [key: string]: any}[] [] 数据源，每项需包含唯一 id
renderItem (item: T, index: number) => ReactNode - 渲染每一项的函数
current number - 当前索引（受控模式）
defaultCurrent number 0 默认索引（非受控模式）
onChange (current: number, item: T) => void - 索引变化回调
itemWidth number 750 每项宽度（单位 rpx）
threshold number 0.1 切换阈值比例（0-1）
damping number 1 阻尼系数
edgeDamping number 0.3 边缘阻尼系数
duration number 300 过渡动画时长（ms）
className string - 自定义类名
style CSSProperties - 自定义样式
